    (1) The reason of choosing the file format when creating your audio

    My files are using mp3 format. I have a source audio file, and I added vocal audio and mixed them as my modified audio. I chose the mp3 format because my audio files are music. And mp3 is the most popular format for music to be played on mobile devices, PCs, and web browsers. Although mp3 is lossy compressed so that it loses some quality, it saves a lot of space and provides better performance for the web browser to load and perform it. And it is really hard for general humans to notice the difference between lossless compressed format such as FLAC and WMA format with mp3 because humans can't hear all sounds in all contexts.

    (2) The use of audio on your website regarding usability.

    I provided two audio files and I embedded them into the HTML. So that the user can see them and control the music using the control components provided by HTML. The users can easily play/stop the music and adjust the volume. Besides the play control function, I also provided links to the audios to allow the user to download or save the audios. So the users can easily receive, control and use my web page without facing difficulties.

    (3) Any thoughts on different audio file formats, e.g., cons, pros, when to use.

    The audio files include uncompressed formats like Wav, Aiff, lossless compression formats like FLAC, WMA, and the most popular lossy compression formats such as mp3, Ogg, ACC, M4A. Nowadays, uncompressed and lossless compressed audio is getting more and more popular due to the growth of hardware performance. However, the lossy compressed is still the best choice to use on web applications. Although the lossy compressed audio cuts off some parts during the compression, the file size is usually small. For example, the file size of a general 4 minutes long song is only around 3MB to 5MB. But if using FLAC, it would be 45MB. If using lossless compressed audio on a web page as background music or something else, it will need more memory space to store the page and a longer time to load. And due to the quality of the users' speakers and humans' hearing capability, the general people are hard to notice the quality difference between FLAC and mp3 with 320kbps. So, when developers need performance, lossy compression formats like mp3 are always the best choice.

So a song mp3 was created. Then I embedded the mp3 file into my HTML in audio tags and set up the controls attribute, finally the files are playable on my website.
